计算机学院毕业设计（论文）任务书
题目：基于 SSE 框架实现校园安全教育平台
类型：应用型
1、研究目标
1.掌握 SSE（Server-Sent Events）框架的核心原理（如 HTTP 长连接机制、事件流格式、自动重连与状态恢复机制），实现服务器向客户端的校园安全预警信息、通知消息实时推送功能，确保消息传输延迟≤2 秒。
2.熟练运用 SpringBoot 与 Vue.js 构建前后端分离架构，完成校园安全教育平台的全栈开发，掌握需求分析、系统设计、编码实现、测试优化的完整开发流程。
3.实现安全知识管理、实时预警、在线测评、应急演练预约等核心功能模块，保障系统在千级并发连接下稳定运行，服务器资源消耗较传统轮询机制降低 80% 以上，提升校园安全教育的实效性与管理效率。
2、主要研究内容和方法
2.1 核心研究内容
结合 MVC 架构模式与指定技术栈，围绕校园安全教育平台的核心功能展开全流程开发，具体研究内容如下：
1.需求分析与 MVC 架构适配设计梳理校园安全教育平台的功能性需求（首页展示、安全课程学习、积分商城、试卷列表、新闻资讯、个人中心、后台管理）与非功能性需求（兼容性、稳定性、易用性），基于 MVC 架构模式拆分模块：模型层（Model）：设计 MySQL8.0 数据库表结构（用户表、课程表、积分表、试卷表、资讯表、管理员表等），通过 MyBatis 实现数据持久化操作，封装业务逻辑（如课程学习进度记录、积分计算规则、试卷评分逻辑）；
a.视图层（View）：基于 Vue、HTML、jQuery 开发前端页面，实现首页可视化展示、课程学习界面、积分商城交互、试卷作答页面、资讯浏览、个人中心与后台管理界面的渲染；
b.控制层（Controller）：通过 Spring MVC 设计请求接口，接收前端请求（如课程学习请求、积分兑换请求、试卷提交请求），调用模型层处理业务逻辑，返回数据至视图层完成页面更新。
2.核心功能模块开发
a.首页模块：基于 Vue+jQuery 实现首页布局，整合轮播图（安全资讯推荐）、课程入口、积分商城入口、试卷列表入口等核心功能入口，适配不同终端的页面响应式展示；
b.安全课程学习模块：设计课程分类（消防安全、交通安全、网络安全等）、课程播放、学习进度记录、学习完成积分奖励等子功能，通过 Spring+MyBatis 实现课程数据的增删改查，前端通过 Vue 实现课程视频 / 图文的加载与学习状态同步；
c.积分商城模块：设计积分规则（课程学习、试卷作答获取积分）、商品列表、商品兑换、兑换记录等子功能，基于 MySQL8.0 存储积分数据与商品数据，通过 Controller 层实现积分增减、商品兑换的业务逻辑校验；
d.试卷列表模块：实现试卷分类、试卷作答、自动评分、成绩记录等功能，通过 MyBatis 实现试卷题库数据的查询与作答结果的存储，Spring 控制层处理试卷提交与评分逻辑，Vue+jQuery 实现作答界面交互与成绩展示；
e.新闻资讯模块：开发资讯发布、分类、浏览、搜索功能，后台管理端支持资讯的增删改查，前端通过 Vue 实现资讯列表分页展示与详情页渲染；
f.个人中心模块：实现用户信息查看 / 修改、学习记录查询、积分明细、考试成绩查询等功能，基于 Spring MVC 实现用户数据的权限校验与数据返回；
g.后台管理模块：针对管理员角色设计用户管理、课程管理、积分商城管理、试卷管理、资讯管理等功能，通过 Spring 实现权限控制，Vue+jQuery 开发管理界面，确保管理员对平台数据的全生命周期管控。
3.技术栈整合与环境适配
a.基于 Java JDK1.8、IDEA 开发工具搭建项目工程，整合 Spring+MyBatis 框架，配置 Tomcat 服务器实现项目部署，确保框架间的兼容性与运行稳定性；
b.适配 MySQL8.0 数据库特性（如事务机制、索引优化），通过 Navicat 完成数据库可视化设计与数据管理，优化 SQL 语句提升数据查询效率；
c.实现前后端数据交互：前端 Vue/jQuery 通过 AJAX 调用 Spring MVC 控制层接口，完成请求发送与数据接收，确保首页、课程学习、积分兑换等功能的实时数据交互。
4.系统测试与优化针对各功能模块开展单元测试、集成测试与功能测试：功能测试：验证首页展示、课程学习、积分兑换、试卷作答、后台管理等核心功能的完整性与准确性；
a.性能测试：优化 MyBatis 查询语句、数据库索引，提升高并发场景下（如多用户同时学习课程、作答试卷）的系统响应速度；
b.兼容性测试：确保前端页面在主流浏览器中正常渲染，适配 MVC 架构下前后端交互的稳定性。
2.2 研究方法
1.文献研究法：查阅 MVC 架构模式、Spring+MyBatis 整合、Vue 前端开发、MySQL8.0 优化等相关文献与技术文档，明确技术实现路径与最佳实践；
2.需求调研法：通过校园师生访谈、问卷调查等方式，明确安全教育平台的功能需求与使用场景，为功能模块设计提供依据；
3.模块化开发法：遵循 MVC 架构拆分模型、视图、控制层，按 “首页→课程学习→积分商城→试卷列表→新闻资讯→个人中心→后台管理” 的顺序分模块开发，降低模块间耦合度；
4.测试驱动法：在各模块开发完成后，通过黑盒测试、白盒测试验证功能正确性，针对测试中发现的问题优化代码与数据库设计；
5.迭代优化法：结合开发过程中的问题反馈，迭代优化前端交互体验、后端业务逻辑与数据库性能，确保系统满足校园安全教育的实际使用需求。
3、主要考核或指标
1.功能完整性：完成实时预警、安全知识管理、在线测评、应急处理等核心模块开发，支持管理员发布预警消息、师生接收并查看预警内容，安全知识检索准确率≥95%，在线测评功能无答题数据丢失或判分错误问题。
2.实时性与性能：SSE 实时推送模块响应延迟≤1 秒，支持同时在线用户≥1000 人，并发场景下服务器 CPU 使用率≤60%、内存占用≤800MB，消息无丢失或重复推送；系统连续运行 72 小时无崩溃，故障恢复时间≤5 分钟。
3.兼容性与安全性：系统兼容 Chrome 90+、Firefox 88+、Edge 90+、IE11（通过 Polyfill 适配）等主流浏览器；实现基于 JWT 的身份认证与 RBAC 权限控制，确保不同角色仅访问授权资源；数据传输采用 HTTPS 加密，用户密码加盐哈希存储，无数据泄露风险。
4.文档规范性：提交完整的技术文档，包括需求分析报告、系统架构设计图（含 SSE 通信流程）、数据库 ER 图、核心代码注释（关键模块注释覆盖率≥90%）、测试报告（含功能测试用例、性能测试数据），文档逻辑清晰、符合软件工程规范。
