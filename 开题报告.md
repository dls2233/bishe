一、选题背景（综述本课题研究现状、选题目的及意义）
  研究现状与背景随着高校信息化建设的深入，校园安全教育已成为学校管理工作的重中之重。传统的校园安全教育往往依赖线下讲座、纸质宣传册或简单的通知发布，存在信息传递滞后、覆盖面有限、学生参与度低以及缺乏实时互动等问题。在突发安全事件（如火灾、极端天气、公共卫生事件）面前，传统的信息发布机制（如轮询方式）往往无法满足“秒级”触达的需求，导致预警信息滞后 。
  当前，虽然部分高校已部署了安全教育系统，但多数系统仍采用传统的 Web 轮询机制来获取消息，服务器资源消耗大且延迟高。如何利用新兴的 Web 实时通信技术（如 SSE, Server-Sent Events）来构建低延迟、高并发、低能耗的安全预警平台，成为当前校园信息化建设的重要研究方向 。此外，现有的平台往往功能单一，缺乏将“知识学习”、“在线测评”、“积分激励”与“应急演练”深度融合的综合性解决方案。

二、研究目标和内容
2.1 研究目标本课题主要致力于达成以下目标：
核心技术攻关：深入掌握并应用 SSE 框架核心原理，实现服务器向客户端的实时消息推送，确保传输延迟≤2 秒，并具备自动重连与状态恢复能力 。
全栈开发实践：熟练运用 SpringBoot 后端框架与 Vue.js 前端框架，构建高可用、易维护的前后端分离架构系统 。
功能完备性：成功实现安全知识管理、实时预警、在线测评、积分商城、应急演练预约等核心业务模块 。
系统高性能：通过数据库索引优化与代码重构，保障系统在千级并发场景下的稳定性，显著降低服务器资源占用 。
2.2 主要研究内容本课题将结合 MVC 架构模式 ，围绕校园安全教育的核心需求展开，具体内容如下：
1. 系统架构设计
模型层 (Model)：基于 MySQL 8.0 设计数据库，包含用户表、课程表、积分表、试卷表、资讯表等核心表结构。使用 MyBatis 实现数据持久化，封装积分计算、试卷评分等核心业务逻辑 。
视图层 (View)：采用 Vue.js + HTML + jQuery 开发前端交互界面，包括响应式的首页、课程学习页、积分商城、试卷作答页及后台管理系统 。
控制层 (Controller)：利用 Spring MVC 设计 RESTful API 接口，处理如课程请求、积分兑换、试卷提交等业务请求，并协调 Model 层与 View 层的数据交互 。
2. 核心功能模块开发
首页模块：整合轮播图（安全资讯）、课程入口、积分商城入口，实现多终端响应式布局 。
安全课程学习模块：支持消防、交通、网络等安全课程分类展示与视频/图文学习，记录学习进度并自动发放积分奖励 。
积分商城模块：设计积分获取与消耗规则，实现商品列表展示与兑换记录管理，包含严格的业务逻辑校验 。
试卷列表与在线测评：建立试卷题库，实现自动评分、成绩记录与错题回顾功能 。
新闻资讯模块：支持安全资讯的发布、分类检索与详情浏览 。
个人中心模块：提供个人信息修改、学习记录查询、积分明细查看及考试成绩归档功能 。
后台管理模块：为管理员提供用户权限管理、课程资源维护、试卷库管理及资讯发布的全权限控制功能 。
3. 技术整合与优化
基于 JDK 1.8 与 IDEA 搭建开发环境，整合 Spring + MyBatis 框架 。
针对 MySQL 8.0 特性进行索引优化与事务管理，提升查询效率 。
实现前后端通过 AJAX/Axios 进行异步数据交互，确保页面无刷新更新。
三、研究方案
3.1 研究方法文献研究法：查阅关于 MVC 架构、Spring+MyBatis 整合、Vue 开发及 SSE 技术的文献与文档，明确技术路径 。
需求调研法：通过访谈与问卷调查，明确校园师生对安全教育平台的具体功能需求（如首页展示偏好、积分规则接受度等） 。
模块化开发法：遵循 MVC 架构，按照“首页→课程→商城→试卷→资讯→个人中心→后台”的顺序分模块独立开发，降低耦合 。
测试驱动法：在开发过程中穿插黑盒与白盒测试，验证功能正确性；通过性能测试优化 MyBatis 查询语句与数据库索引 。
3.2 实施步骤准备阶段：搭建 Java JDK、IDEA、MySQL、Node.js 等开发环境，完成相关技术文档的阅读与 Demo 验证 。
分析与设计阶段：梳理功能性与非功能性需求，完成数据库 ER 图设计与 MVC 架构分层设计，输出需求规格说明书 。
编码实现阶段：
后端：搭建 SpringBoot 工程，配置 MyBatis 与 MySQL 连接，开发 Controller 与 Service 层接口 。
前端：搭建 Vue 脚手架，开发各功能页面的 UI 组件，并对接后端接口 。
核心攻关：重点实现 SSE 消息推送模块，确保预警信息的实时性 。
测试与优化阶段：进行单元测试与集成测试，模拟多用户并发场景（如 1000 人同时在线），优化 SQL 语句与服务器配置，确保资源消耗达标 。
部署与总结阶段：配置 Tomcat 服务器进行项目部署，整理开发文档、测试报告，撰写毕业论文 。
3.3 拟解决的主要问题及措施问题：高并发下的实时消息推送延迟
措施：放弃传统的轮询机制，深入研究并采用 SSE（Server-Sent Events）长连接技术，维持服务器与客户端的持久连接，大幅降低握手开销与延迟 。
问题：大量数据查询导致的响应缓慢
措施：针对 MySQL 8.0 特性进行优化，对用户表、课程表等高频查询表建立合理的索引；在 MyBatis 层优化 SQL 语句，避免全表扫描 。
问题：前后端分离架构下的跨域与状态管理
措施：后端配置 CORS 策略允许跨域访问，前端使用 Vuex 进行状态管理，利用 JWT (JSON Web Token) 实现无状态的身份认证与权限控制 。
